Getting started with our APIs

Learn about what APIs we offer and how you can connect to them. These simple steps will get your team building in no time.

Overview

If you are building justice sector software, and need to integrate with HMCTS APIs, this guide is for you.

Whether you’re a product owner, delivery lead, architect, software developer or a business analyst, you’ll have a part to play in this process.

If you own an API and want to list it on the Marketplace, see Producer Standards instead.

Steps to get connected

  1. Learn about our APIs

    Before designing your software, read the Onboarding guide, browse the API Catalogue, and check the Consumer Guidance for rate limits and error handling. Review the Data Governance Standards for the data classification and legal obligations that apply. All APIs use OAuth 2.0 with JWT bearer tokens.

  2. Create a developer account

    Create a developer account on the Marketplace and tell us which APIs you’re interested in. You’ll be able to manage your access requests, track approvals, and get issued sandbox and production credentials once approved.

  3. Confirm your use case

    For APIs that return personal data, confirm you have a valid use case and, where required, a Data Sharing Agreement (DSA) or Data Processing Agreement (DPA) in place. Do this before you go too far with development — do not request access speculatively.

    Required for personal data APIs

  4. Design and build your software

    Refer to the API documentation in the catalogue as you build, and get help and guidance on using APIs responsibly.

  5. Test your software

    Use the sandbox environment to develop and test your integration. Never test against production. Sandbox credentials are issued once your access request is approved.

  6. Get your access approved

    Submit an access request describing your use case, expected call volume and environment. The API owner reviews requests within 2 working days.

    2 working days

    Request API access →

  7. Put your software live

    Once your access request is approved and you’ve confirmed your integration works in sandbox, request production credentials and go live.

Get help and support

If you have any issues, either during development or once your software is live, you can contact the marketplace team.

Read the Consumer Guidance for detailed information on using APIs responsibly, or the Data Governance Standards for your legal obligations.
